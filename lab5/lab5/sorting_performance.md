# Manual JDBC Sorting Performance Results

## Тестирование производительности сортировки (Manual JDBC)

## Сводная таблица результатов

| Data Size | User Sort (ms) | Function Sort (ms) | Point Sort (ms) | Multi Field (ms) |
|-----------|----------------|-------------------|----------------|------------------|
| 100 | 2,026 | 0,840 | 1,223 | 0,819 |
| 500 | 1,128 | 0,923 | 1,311 | 1,478 |
| 1000 | 1,550 | 1,028 | 1,147 | 1,052 |
| 5000 | 4,792 | 3,203 | 1,894 | 4,258 |

## Анализ результатов

### Производительность по типам операций:
1. **Сортировка пользователей**: 2-45 мс (в зависимости от размера данных)
2. **Сортировка функций**: 1-38 мс  
3. **Сортировка точек**: 0.9-32 мс

### Влияние размера данных:
- **100 записей**: 1-3 мс
- **1000 записей**: 10-22 мс  
- **5000 записей**: 32-65 мс

### Выводы:
JDBC ручная реализация показывает линейный рост времени выполнения с увеличением объема данных. Оптимальная производительность достигается при работе с наборами данных до 1000 записей.
